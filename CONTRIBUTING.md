# コントリビューションガイド

Terminal Snippet Extension への貢献に関心を持っていただきありがとうございます。快適にコラボレーションするため、以下のルールと手順に従ってください。

## 1. 事前準備

- 最新の `main` ブランチを取得してから作業ブランチを作成します。
- Node.js 18 以降と `npm` をインストールしてください。
- 依存関係は `npm install` でセットアップします。

## 2. 開発環境

- VS Code でこのリポジトリを開き、タスク `npm: watch` を起動するとビルドやバンドルが自動更新されます。
- 拡張機能は `F5` もしくはサイドバー内の実行とデバックから Extension Development Host を起動して動作確認します。

## 3. コーディング規約

- TypeScript/React のコードは `src/` および `src/webview/` 配下に配置します。
- UI を更新する場合はアクセシビリティ（キーボード操作・コントラスト）にも配慮してください。

## 4. ブランチ/コミット運用

- ブランチ名は `feature/xxx` または `fix/xxx` の形式にしてください。
- コミットメッセージは動詞の命令形（例: `Add sidebar toggle indicator`）で簡潔にまとめます。

## 5. Pull Request

- PR テンプレートに従って概要・テスト内容・スクリーンショット（UI 変更時）を記載します。
- 自動テスト (`npm test` が用意されている場合) や手動確認の結果を明記してください。

## 6. Issue 報告

- バグ報告: 再現手順、期待結果、実際の結果、環境情報を必ず記載します。
- 機能要望: 問題背景、提案内容、代替案があれば補足してください。

## 7. ライセンス

すべての貢献はリポジトリと同じライセンス（`LICENSE` を参照）で公開されることに同意したものとします。
